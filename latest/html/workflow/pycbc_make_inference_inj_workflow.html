pycbc_make_inference_inj_workflow: A parameter estimation workflow generator for injections¶

Introduction¶

The executable pycbc_make_inference_inj_workflow is a workflow generator to setup a parameter estimation analysis.

Workflow configuration file¶

A sample workflow configuration file:

[workflow]
; basic information used by the workflow generator
file-retention-level = all_triggers
h1-channel-name = H1:LOSC-STRAIN
l1-channel-name = L1:LOSC-STRAIN
start-time = 1126259452
end-time = 1126259468


[workflow-ifos]
; the IFOs to analyze

                     '"secondary_mass(mass1,mass2):mass2"'

                     '"primary_spin(mass1,mass2,spin1_azimuthal,spin2_azimuthal):spin1_azimuthal"'
                     '"primary_spin(mass1,mass2,spin1_polar,spin2_polar):spin1_polar"'

Generate the workflow¶

To generate a workflow you will need your configuration files. Generate the workflow using following example run script:

#! /bin/bash

set -e
workflow_name=emcee_pt_pptest
output_dir=run

export OMP_NUM_THREADS=1

pycbc_make_inference_inj_workflow \
    --output-dir ${output_dir} \
    --workflow-name ${workflow_name} \
    --data-type simulated_data \
    --output-file ${workflow_name}.dax \
    --inference-config-file inference.ini \
    --config-files workflow_config.ini \
    --config-overrides results_page:output-path:${PWD}/${output_dir}/results_html

echo "Now run:"
echo "cd ${output_dir}"
echo "pycbc_submit_dax --no-create-proxy --no-grid --dax ${workflow_name}.dax --accounting-group ${accounting_group} --enable-shared-filesystem"

Plan and execute the workflow¶

If you are on LDG, you need to define an accounting group. Finally plan and submit the workflow with:

# submit workflow
cd ${output_dir}
pycbc_submit_dax --dax ${WORKFLOW_NAME}.dax \
    --no-grid \
    --enable-shared-filesystem \
    --accounting-group ${ACCOUNTING_GROUP}
